# Protocol: Source Grounding

## Purpose

Ground learning in a named source before interpreting, discussing, or internalizing it.

This protocol applies when the learner refers to a book, chapter, article, video, podcast, course, note, or reading trace.

## When to Use

- The learner says they read, watched, listened to, or saved something.
- The learner shares a passage, link, screenshot, note, highlight, or reading trace.
- The learner wants to discuss a source rather than a standalone concept.
- The system may otherwise drift into generic explanation.

## Workflow

1. Identify the source and the learner's relationship to it.
2. Check what context is available:
   - full source text
   - excerpt or transcript
   - user notes
   - reading traces
   - user summary only
3. Assign a grounding level:
   - full source + reading traces
   - full source only
   - reading-trace grounded
   - user-led exploration
   - no usable source context
4. Build a lightweight source map when context allows:
   - core claim or question
   - main structure
   - key concepts
   - examples or evidence
   - likely friction points
5. Route to the smallest useful Skill set.
6. Keep later claims aligned with the grounding level.

## Fallbacks

If the source cannot be accessed, ask for a passage, summary, notes, screenshots, transcript, or the learner's remembered point.

If the learner cannot provide context, continue only as user-led exploration and say so briefly.

## Outputs

- Grounding level.
- Lightweight source map.
- Source-specific learning route.
- Open source areas not yet covered.

## Compatible Skills

- Deep Dive
- Review
- Synthesis
- Application
- Reflection

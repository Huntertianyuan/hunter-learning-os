# Protocol: Learning Closure

## Purpose

Close meaningful learning sessions in a way that carries understanding forward.

Learning Closure prevents a session from ending with only a pleasant insight or report.

It helps decide whether to test understanding, connect to life, create Memory, or prepare future review.

It should also produce effective feedback when a session has enough substance to carry forward.

Effective feedback separates what the learner learned from how the learner's thinking changed.

## When to Use

- A concept has been sufficiently grounded and discussed.
- A source discussion reached a meaningful insight or stopping point.
- A synthesis created a useful cross-domain connection.
- The learner asks to pause, summarize, remember, review, or end.
- The session produced a candidate cognitive change, concept map, or practice.

## When NOT to Use

- The exchange was purely factual or very short.
- The learner is still missing basic orientation.
- The learner explicitly wants to stop without summary.
- Closure would add burden without future value.

## Closure Moves

Choose the smallest useful set.

Do not force every move every time.

### 1. Feynman Check

Use when the learner has enough grounding to explain the idea.

Ask for a simple explanation, often:

```text
Try explaining this to a smart 12-year-old.
```

Evaluate whether the explanation:

- avoids unexplained jargon
- includes a concrete example
- preserves the core mechanism
- distinguishes the concept from a nearby confusion

### 2. Application Bridge

Use when the concept or insight may change perception, judgment, or action.

Useful prompts:

- What does this help you notice in real life?
- What problem would you now examine differently?
- What would change if you used this idea this week?

### 3. Memory Candidate

Use when understanding, self-view, or concept structure shifted.

Prefer:

- Cognitive Change History for changed mental models
- Concept History for clarified concepts
- Idea Network for cross-domain connections
- Practice and Review Items for follow-up actions

### 4. Review Prompt

Use when the material is worth retaining.

Good review prompts test understanding, not wording.

Examples:

- Explain the concept simply.
- Give one real-life example.
- Distinguish it from a nearby concept.
- Explain how it changed your view of a familiar phenomenon.

### 5. Coverage Map

Use when a source, concept, or synthesis has several branches.

Show:

- what was understood
- what remains unclear
- useful connections
- possible next routes

### 6. Effective Feedback

Use after meaningful learning sessions, especially when the learner asks to end, save, review, or understand what they gained.

Default format:

```text
Knowledge Gain:
- Concepts, source points, distinctions, examples, origins, or structures clarified.

Cognitive Gain:
- Changed understanding, changed frame, changed real-life interpretation, or changed attention.
- If no visible shift occurred, say: "No clear Cognitive Gain yet."

Worth Saving:
- Memory Type:
- Save Target:
- Why it matters:

Review Queue:
- Recall prompt:
- Recall type:
- Recall strength:
- Suggested interval:

Future Use:
- How this may help review, synthesis, application, or monthly reflection.
```

Do not force Cognitive Gain.

If the session only clarified source content or concepts, record Knowledge Gain only and create knowledge review prompts.

## Default Closure By Session Type

### Concept Onboarding

Usually consider:

- Feynman Check
- Application Bridge
- Concept History candidate
- 1-3 review prompts
- Effective Feedback with Knowledge Gain first

### Source-based Learning

Usually consider:

- source coverage map
- Effective Feedback
- Knowledge Gain for source points discussed
- Cognitive Change candidate only if understanding shifted
- review prompts if retention matters

### Synthesis

Usually consider:

- what connection became clearer
- Idea Network candidate
- boundary question to avoid flattening differences
- Effective Feedback showing both connection learned and frame changed

### Application

Usually consider:

- one concrete practice
- follow-up review question
- optional Practice and Review Item
- Effective Feedback showing what knowledge became usable

### Reflection

Usually consider:

- Cognitive Change candidate
- open question
- optional future check-in prompt
- Effective Feedback, without inventing knowledge or cognitive gains

## Periodic Review Direction

This protocol prepares review-ready outputs.

It does not automate reminders.

Future weekly or monthly reviews should summarize:

- concepts encountered
- concepts recallable versus fuzzy
- cognitive changes
- idea network links
- practices tried
- open questions

Knowledge-only review items should be reviewed through active recall:

- source recall
- content recall
- distinction recall
- application recall

Recall strength:

```text
0 = not remembered
1 = vaguely remembered
2 = basically clear
3 = clear and can give an example or application
```

Suggested intervals:

```text
0 -> 1 day
1 -> 3 days
2 -> 7 days
3 -> 14-30 days
```

## Outputs

- Closure checkpoint.
- Simple explanation attempt or prompt.
- Application bridge.
- Effective feedback.
- Memory candidate.
- Review prompts.
- Coverage map.

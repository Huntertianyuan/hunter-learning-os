# Skill: Application

## Purpose

Translate understanding into real-world judgment, action, practice, or decision-making.

Application is for making knowledge usable outside the conversation.

## When to Use

- The user asks how an idea applies to life, work, relationships, learning, or decisions.
- The user wants a practice, experiment, or observation task.
- The user understands a concept but does not know how to use it.
- A source insight should change behavior or perception.
- The user is facing a real situation that can be reframed by a concept.

## When NOT to Use

- The concept is not yet understood.
- The user wants memory review rather than action.
- The user wants open reflection without practical next steps.
- The situation is too vague to apply anything responsibly.
- The issue requires professional help rather than self-guided practice.

## Default Method Sequence

1. `distinction`: clarify what exactly should be applied.
2. `transfer`: map the idea from source context to real context.
3. `counterexample`: test whether the application is too broad or forced.
4. `inversion`: identify failure modes and hidden risks.
5. `practice-design`: create a small observation, exercise, or experiment.
6. `metacognition`: notice how the application changes the user's thinking or behavior.

## Optional Method Add-ons

- `first-principles`: when the decision needs foundational reframing.
- Scenario testing behavior is handled through `transfer` and `practice-design` in v0.1.
- `concept-mapping`: when several concepts shape one situation.
- `socratic-questioning`: when the user has a hidden assumption about the situation.

## Memory Outputs

- Real-life application examples.
- Practice candidates.
- Decision reframes.
- Observed behavior or perception shifts.
- Cognitive Change History candidates when an idea changes action.

## Exit Conditions

- The user has a concrete interpretation, action, practice, or next observation.
- The application has been tested for realism.
- The concept turns out not to apply and the system returns to Deep Dive or Synthesis.
- The user chooses not to turn the insight into action now.

## Example Scenarios

- "How can I use this chapter in my anxiety about money?"
- "Design a small practice for observing attachment."
- "How does this idea apply to my work?"
- "Can game theory help me think about negotiation?"
- "What should I do differently if I accept this concept?"

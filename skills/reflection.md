# Skill: Reflection

## Purpose

Track cognitive growth, mental model changes, self-understanding, and learning patterns.

Reflection is for noticing how the learner's thinking changes over time.

## When to Use

- The user notices a change in their understanding.
- The user wants to examine their own pattern, reaction, belief, or learning process.
- A discussion reveals a recurring cognitive or emotional theme.
- The system needs to identify a Cognitive Change History candidate.
- The user wants to pause, summarize, or understand what just happened in the conversation.

## When NOT to Use

- The user wants direct concept explanation.
- The user primarily wants to remember material.
- The user wants practical action more than self-observation.
- Reflection would become intrusive, over-personal, or ruminative.
- The learner wants to internalize a source or concept, not receive psychological counseling.
- The user has signaled they do not want to go deeper.

## Default Method Sequence

1. `question-driven-learning`: name the reflective question.
2. `metacognition`: identify the user's thinking pattern or change.
3. `distinction`: separate source understanding from self-understanding.
4. `socratic-questioning`: gently examine important assumptions if the user wants depth.
5. `practice-design`: create a small observation task when useful.
6. `progressive-summarization`: close with a concise reflection.

## Optional Method Add-ons

- `first-principles`: when a deep belief or rule needs examination.
- `inversion`: when a pattern is easier to understand through its failure mode.
- `transfer`: when reflection should connect to real situations.
- `active-recall`: when reviewing prior reflections.
- `synthesis`: when the reflection connects multiple life themes or learning threads.

## Memory Outputs

- Cognitive Change History candidates.
- Learning Profile updates.
- Recurring patterns, beliefs, and open questions.
- Practice follow-ups.
- Before/after understanding records.

## Exit Conditions

- A meaningful cognitive movement has been named.
- The user has an open question or practice to carry forward.
- The conversation risks becoming too probing and should pause.
- The user chooses to save, summarize, shift, or stop.
- The reflection reveals a need for Deep Dive, Application, or Review.
- Two consecutive personal probing questions have already been asked without explicit consent to continue.

## Example Scenarios

- "I used to think happiness came from material conditions, but now I am not sure."
- "I notice I keep chasing the next goal."
- "Can we summarize what changed in my understanding?"
- "Why did this passage bother me so much?"
- "Help me reflect on why I procrastinated today."

# Method: Practice Design

## Purpose

Turn an insight into a small practice, experiment, or observation task.

This method helps learning leave the conversation and continue in real life.

## When to Use

- The user has reached an insight but needs a way to live with it.
- A concept should be tested in daily life.
- The user wants actionable learning without turning it into productivity pressure.
- A session needs a concrete next step.

## When NOT to Use

- The insight is not clear enough yet.
- The user does not want action or homework.
- The practice would be too ambitious, vague, or guilt-producing.
- The issue requires professional support rather than self-directed practice.

## Inputs

- Insight or concept.
- User's real-life context.
- Desired observation or behavior.
- Time frame and effort level.

## Workflow

1. Name the insight to practice.
2. Choose a small real-life trigger or context.
3. Design a low-friction observation, question, or action.
4. Define how the user will know they did it.
5. Keep the practice short and optional.
6. Decide whether to review it later.

## Outputs

- Practice prompt.
- Observation task.
- Small experiment.
- Optional review question.

## Success Criteria

- The practice is small enough to actually do.
- It connects directly to the insight.
- It increases awareness, behavior change, or future recall.
- It does not create unnecessary pressure.

## Memory Integration

- Save practices as follow-up candidates.
- Link practice results to Cognitive Change History.
- Update Learning Profile with practices that work for the user.
- Convert repeated practices into stable learning patterns.

## Compatible Skills

- Application
- Reflection
- Review
- Deep Dive

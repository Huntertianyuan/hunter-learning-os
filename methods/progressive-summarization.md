# Method: Progressive Summarization

## Purpose

Compress material through layers, from source detail to core meaning and usable understanding.

This method turns long or complex material into something the learner can remember, revisit, and apply.

## When to Use

- The user wants to remember a chapter, article, lecture, or conversation.
- The source has many details and needs compression.
- The user needs a clear takeaway without losing structure.
- A session should end with a concise record.

## When NOT to Use

- The user is still exploring and compression would close the inquiry too early.
- The user needs exact source grounding or quotation.
- The material is already short and clear.
- The goal is creative generation rather than condensation.

## Inputs

- Source material or discussion transcript.
- User's highlighted points or questions.
- Key concepts and structure.
- Desired level of compression.

## Workflow

1. Identify the source's main structure.
2. Extract important claims, examples, and distinctions.
3. Compress into a short summary.
4. Compress again into a core model, sentence, or set of takeaways.
5. Add the user's personal understanding or unresolved question.
6. Mark what should be remembered or revisited.

## Outputs

- Layered summary.
- Core takeaway.
- Memorable model or phrase.
- Review-ready notes.

## Success Criteria

- The summary preserves the source's central meaning.
- The user can use the compressed version to recall the larger material.
- The user's own understanding is represented.
- The summary is short enough to revisit.

## Memory Integration

- Save compressed takeaways to Concept History.
- Create review anchors for Active Recall or Spaced Repetition.
- Add personal takeaways to Cognitive Change History candidates.
- Link summarized concepts to the Idea Network.

## Compatible Skills

- Deep Dive
- Review
- Synthesis
- Reflection

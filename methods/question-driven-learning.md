# Method: Question-Driven Learning

## Purpose

Use the learner's question as the engine of learning.

This method prevents learning from becoming passive content consumption. It turns a source, concept, or situation into an inquiry shaped by what the learner is actually trying to understand.

## When to Use

- The user has a real question, confusion, curiosity, doubt, or tension.
- The user says they want to "discuss" something but has not chosen a precise angle.
- A source contains too much material and the system needs a learning focus.
- The user is likely to learn more by pursuing a question than by receiving a summary.

## When NOT to Use

- The user explicitly asks for a direct factual answer or concise summary.
- The user wants memorization or review rather than exploration.
- The source must first be grounded before any question can be answered responsibly.
- The user has already chosen a clear Method path, such as Active Recall or Practice Design.

## Inputs

- User's stated question or implicit concern.
- Source context, if available.
- Reading traces, if available.
- Relevant memory about prior interests, misunderstandings, or recurring themes.

## Workflow

1. Identify the user's explicit question.
2. If no clear question is stated, infer 1-3 possible live questions from the user's wording.
3. Reflect the question back in plain language.
4. Choose the most fruitful starting question with the user, or proceed with the strongest inferred question if the path is obvious.
5. Use the question to select supporting Methods, such as Distinction, Socratic Questioning, Analogy, or Transfer.
6. Periodically check whether the original question has changed.
7. End by naming the current answer, the remaining open question, or the next question that emerged.

## Outputs

- A clear learning question.
- A focused learning path.
- A record of how the question changed during the session.
- Candidate follow-up questions.

## Success Criteria

- The user feels the discussion is addressing their real curiosity.
- The conversation does not drift into generic explanation.
- The question becomes clearer, deeper, or more answerable.
- The user can state what they are now trying to understand.

## Memory Integration

- Update Learning Profile with recurring question patterns.
- Update Concept History when a question reveals unclear or emerging concepts.
- Create a Cognitive Change History candidate if the user's question changes meaningfully.
- Preserve strong open questions as future retrieval anchors.

## Compatible Skills

- Deep Dive
- Synthesis
- Application
- Reflection

# Method: Distinction

## Purpose

Separate concepts, motives, feelings, or claims that are being mixed together.

This method reduces confusion by making boundaries visible.

## When to Use

- The user is using one word to cover multiple meanings.
- Two concepts are close but not identical.
- The user feels stuck because different issues are tangled.
- A source makes a subtle conceptual distinction that matters.

## When NOT to Use

- The user needs an example or story before abstract distinctions will land.
- The distinction would become pedantic and not improve understanding.
- The user is asking for synthesis rather than separation.
- The concepts are already clear enough for the current goal.

## Inputs

- The concepts or terms to distinguish.
- User's current wording.
- Source definitions or examples, if available.
- A real context where the distinction matters.

## Workflow

1. Name the mixed concepts.
2. Define each concept in plain language.
3. Show where they overlap.
4. Show where they differ.
5. Apply the distinction to the user's concrete question.
6. Check whether the distinction changes the user's interpretation.

## Outputs

- A clarified conceptual boundary.
- A comparison table or concise contrast, when useful.
- A revised understanding of the user's question.
- Candidate examples for future retrieval.

## Success Criteria

- The user can explain the difference without overcomplicating it.
- The distinction resolves a real confusion.
- The distinction changes how the user reads, judges, or acts.
- The system avoids false dichotomies.

## Memory Integration

- Update Concept History with clarified definitions.
- Record common confusions in Misconception History.
- Add useful distinctions to the user's Idea Network.
- Save distinctions that repeatedly shape the user's thinking.

## Compatible Skills

- Deep Dive
- Synthesis
- Application
- Reflection

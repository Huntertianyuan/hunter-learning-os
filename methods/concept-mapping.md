# Method: Concept Mapping

## Purpose

Represent concepts and their relationships as a structured map.

This method helps the learner move from isolated ideas to an organized understanding.

## When to Use

- A topic contains multiple related concepts.
- The user is confused about how ideas connect.
- The system needs to compare frameworks, arguments, or models.
- A discussion has produced many scattered insights.

## When NOT to Use

- The user only needs one concept explained.
- The relationship between concepts is not yet understood enough to map.
- The map would become decorative rather than clarifying.
- The user is in a reflective mode where structure would interrupt expression.

## Inputs

- List of concepts.
- Relationships among concepts.
- Source structure, if available.
- User's current understanding.

## Workflow

1. Identify the key concepts.
2. Group related concepts.
3. Name relationship types, such as causes, contrasts, examples, layers, or sequences.
4. Create a compact text map.
5. Use the map to locate confusion, gaps, or central ideas.
6. Revise the map as understanding improves.

## Outputs

- A concept map.
- Identified central and peripheral concepts.
- Relationship labels.
- Gaps or unclear links.

## Success Criteria

- The user can see the structure of the topic.
- The map reduces confusion rather than adding complexity.
- The map supports later recall, synthesis, or application.
- The user can modify or explain the map.

## Memory Integration

- Update Idea Network with concept links.
- Add stable concept clusters to Concept History.
- Record unclear links as open questions.
- Use repeated maps to track cognitive growth over time.

## Compatible Skills

- Deep Dive
- Review
- Synthesis
- Reflection

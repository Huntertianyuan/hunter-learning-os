# Method: Transfer

## Purpose

Move an idea from its original context into a new real-world context.

This method helps knowledge become usable beyond the source where it was learned.

## When to Use

- The user asks how a concept applies to life, work, relationships, practice, or decisions.
- The user understands an idea intellectually but has not used it.
- A source insight needs to become action or perception.
- The system wants to test whether understanding is flexible.

## When NOT to Use

- The concept is not yet understood.
- The new context is too vague.
- The application would be forced or superficial.
- The user wants memory review rather than application.

## Inputs

- Source concept.
- Target context.
- User's real situation or imagined scenario.
- Constraints, goals, and stakes.

## Workflow

1. State the source concept clearly.
2. Identify the target context.
3. Map the concept's structure onto the context.
4. Identify what changes and what stays the same.
5. Generate a possible interpretation, action, or decision shift.
6. Test whether the transfer feels real and useful.

## Outputs

- Applied interpretation.
- Real-world example.
- Action or decision implication.
- Transfer limits.

## Success Criteria

- The concept changes how the user sees or acts in a real context.
- The application is specific, not generic.
- The transfer preserves the concept's meaning.
- The user can recognize similar future contexts.

## Memory Integration

- Save successful transfers in Cognitive Change History.
- Link concepts to real-life contexts in Idea Network.
- Update Learning Profile with domains where the user applies ideas.
- Record failed or forced transfers for later refinement.

## Compatible Skills

- Application
- Deep Dive
- Synthesis
- Reflection

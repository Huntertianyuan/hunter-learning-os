# Method: Active Recall

## Purpose

Strengthen memory and test understanding by asking the learner to retrieve information without looking.

This method turns recognition into recall.

## When to Use

- The user wants to remember a concept, chapter, lesson, or procedure.
- The system needs to test what the user can retrieve.
- The user overestimates understanding because the material feels familiar.
- A review session is starting or ending.

## When NOT to Use

- The user has not yet formed an initial understanding.
- The goal is reflection or application rather than recall.
- The recall prompt would be too broad or vague.
- The user is fatigued and likely to experience recall as punishment.

## Inputs

- Target material.
- Prior summaries or memory anchors.
- Desired recall scope.
- Time since last exposure, if known.

## Workflow

1. Hide or set aside the source.
2. Ask the user to recall the core idea, structure, examples, or steps.
3. Compare recall with the source or prior summary.
4. Identify omissions, distortions, and strong points.
5. Create focused follow-up prompts for weak areas.
6. Repeat later through Spaced Repetition if needed.

## Outputs

- Recall attempt.
- Gap list.
- Corrected recall.
- Review prompts.

## Success Criteria

- The user retrieves from memory rather than rereading.
- Gaps become visible.
- Corrected recall improves specificity and structure.
- Future review can target weak points.

## Memory Integration

- Update Concept History with recall strength.
- Add weak items to review queue candidates.
- Track repeated forgetting patterns in Learning Profile.
- Save strong recall prompts for later use.

## Compatible Skills

- Review
- Deep Dive
- Application

# Method: First Principles

## Purpose

Break a problem or belief down to its most basic assumptions and rebuild understanding from there.

This method helps the learner escape inherited frames, slogans, and surface-level reasoning.

## When to Use

- The user is trapped in conventional assumptions.
- A problem feels complex because too many secondary claims are layered together.
- The user wants to understand the foundation of a concept or decision.
- A belief needs to be rebuilt rather than merely adjusted.

## When NOT to Use

- The user needs practical next steps more than foundational analysis.
- The topic requires specialized facts the system does not have.
- The user is asking for emotional reflection, not structural reasoning.
- The first principles exercise would become abstract and disconnected.

## Inputs

- Problem, belief, or concept to analyze.
- Current assumptions.
- Relevant constraints or facts.
- Desired decision or understanding.

## Workflow

1. State the problem or belief.
2. Strip away inherited explanations and secondary assumptions.
3. Identify what must be true at the base level.
4. Separate facts, values, constraints, and interpretations.
5. Rebuild a simpler model from the base assumptions.
6. Compare the rebuilt model with the user's original view.

## Outputs

- Basic assumptions.
- Rebuilt model.
- Removed assumptions or inherited frames.
- Revised understanding or decision frame.

## Success Criteria

- The user sees which assumptions were doing the work.
- The rebuilt model is simpler and clearer.
- The method changes the user's view of the problem.
- The result remains connected to the real question.

## Memory Integration

- Save changed assumptions as Cognitive Change History candidates.
- Update Learning Profile with recurring foundational beliefs.
- Add rebuilt models to Concept History.
- Link first-principles models to future decisions.

## Compatible Skills

- Deep Dive
- Synthesis
- Application
- Reflection

# Method: Metacognition

## Purpose

Help the learner observe how they think, understand, misunderstand, decide, and change.

This method makes cognition itself visible.

## When to Use

- The user is reflecting on their own patterns.
- A discussion reveals a recurring belief, reaction, or learning habit.
- The system needs to distinguish source understanding from self-understanding.
- The user wants cognitive growth rather than only content mastery.

## When NOT to Use

- The user wants a direct explanation or practical answer.
- Reflection would become overanalysis or rumination.
- The user has signaled they do not want to go inward.
- The source itself is not yet understood enough to reflect from.

## Inputs

- User's statements, reactions, questions, and hesitations.
- Source or concept being reflected on.
- Prior memory about recurring patterns, if available.
- Current emotional and conversational boundaries.

## Workflow

1. Notice the user's thinking pattern without overdiagnosing.
2. Reflect it back as a tentative observation.
3. Ask whether it fits or misses.
4. Connect the pattern to the learning topic only if useful.
5. Identify what changed in the user's thinking, if anything.
6. Offer to stop, summarize, apply, or save a memory candidate.

## Outputs

- A named cognitive pattern.
- A before/after understanding, if present.
- An open question about the user's thinking.
- Candidate memory or practice.

## Success Criteria

- The user recognizes something about their own thinking.
- The reflection feels accurate and non-coercive.
- The conversation does not become a questionnaire.
- The insight can inform future learning or action.

## Memory Integration

- Save meaningful shifts as Cognitive Change History candidates.
- Update Learning Profile with recurring cognitive patterns.
- Link self-observations to concepts and practices.
- Preserve open questions that may guide future reflection.

## Compatible Skills

- Reflection
- Deep Dive
- Application
- Synthesis

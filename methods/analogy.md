# Method: Analogy

## Purpose

Use a familiar structure to help the learner understand an unfamiliar or abstract idea.

This method makes difficult concepts more graspable without reducing them to mere slogans.

## When to Use

- The concept is abstract, technical, philosophical, or hard to visualize.
- The user asks for an example but needs a structural comparison.
- The user has a familiar domain that can illuminate the new idea.
- A source uses an analogy that should be unpacked.

## When NOT to Use

- The analogy would oversimplify or distort the concept.
- The user needs precise distinction rather than resemblance.
- The concept is already concrete enough.
- The analogy risks replacing understanding with a catchy image.

## Inputs

- Target concept.
- Familiar domain or experience.
- Key features that must be preserved.
- Known limits of the analogy.

## Workflow

1. Identify the abstract structure to be understood.
2. Choose a familiar comparison with a similar structure.
3. Map each important part of the analogy to the target concept.
4. State where the analogy breaks.
5. Ask the user to use the analogy to explain the concept back or apply it.

## Outputs

- A usable analogy.
- A mapping between analogy and concept.
- Explicit limits of the analogy.
- Improved intuitive understanding.

## Success Criteria

- The user can use the analogy to explain the idea.
- The analogy preserves the core structure.
- The user understands where the analogy fails.
- The analogy helps later recall or application.

## Memory Integration

- Save strong analogies as Concept History anchors.
- Link analogies to the Idea Network.
- Record analogy failures that reveal misconceptions.
- Update Learning Profile with analogy domains that work well for the user.

## Compatible Skills

- Deep Dive
- Review
- Synthesis
- Application
